# Design Debt Backlog

Use this lightweight backlog to track recurring UI issues and batch improvements.

## How to use

1. Add an item when the same class of visual/interaction issue appears repeatedly.
2. Link the PR(s) where the issue was observed.
3. Keep entries short and actionable.
4. Move completed items to the "Done" section with the completion date.
